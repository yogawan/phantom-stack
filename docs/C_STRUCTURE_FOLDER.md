# Project Folder Structure Documentation

This document provides an overview of the folder structure for the project, explaining the purpose and contents of each directory.

## Root Directory
- **.env.local**: Contains environment-specific variables for local development.
- **.gitignore**: Specifies files and directories to be ignored by Git.
- **biome.json**: Configuration file for the Biome tool.
- **components.json**: Configuration file for components.
- **next-env.d.ts**: TypeScript definitions for Next.js.
- **next.config.ts**: Configuration file for Next.js.
- **package.json**: Lists project dependencies and scripts.
- **postcss.config.mjs**: Configuration for PostCSS.
- **README.md**: Project overview and setup instructions.
- **tsconfig.json**: TypeScript configuration file.

## Directories

### `.next`
Contains build artifacts and cache files generated by Next.js.

### `app`
- **globals.css**: Global CSS styles.
- **layout.tsx**: Root layout component for the application.
- **page.tsx**: Default page component.
- **api/**: API route handlers for server-side logic.

### `components`
Organized using Atomic Design principles:
- **atoms/**: Small, reusable components (e.g., `Button`, `Input`).
- **molecules/**: Combinations of atoms (e.g., `Form`, `Card`).
- **organisms/**: Complex UI components (e.g., `Sidebar`, `Table`).
- **templates/**: Page-level layout components.

### `config`
- **database.ts**: Database configuration and connection logic.

### `docs`
Documentation files for the project:
- **A_TECH_STACK.md**: Details the technologies used.
- **B_ATOMIC_DESIGN.md**: Explains the Atomic Design methodology.
- **C_STRUCTURE_FOLDER.md**: This document.
- **D_API_CALL_FLOW.md**: Describes the API call flow.

### `hooks`
Custom React hooks for shared logic:
- **useAuth.ts**: Authentication-related logic.
- **useMobile.ts**: Mobile-specific utilities.
- **useProperty.ts**: Property-related logic.

### `lib`
Utility libraries and helpers:
- **apiClient.ts**: Axios instance for API calls.
- **mongodb.ts**: MongoDB connection logic.
- **utils.ts**: General utility functions.

### `models`
- **User.ts**: Mongoose schema and model for the `User` entity.

### `public`
Static assets such as images and icons.

### `scripts`
- **seed.ts**: Script for seeding the database.

### `services`
Service layer for business logic:
- **auth/**: Authentication services.
- **property/**: Property-related services.

### `stories`
Storybook files for UI component documentation and testing.

### `types`
TypeScript type definitions for the project.

---

This structure ensures modularity, scalability, and maintainability of the codebase.
